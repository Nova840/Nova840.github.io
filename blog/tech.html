Dynamic vs. Static-typed Languages.

4/2/16

Dynamic-typed languages are languages where you don't need to specify a variable's type. Examples of this are languages like JavaScript, Ruby, and Python. These languages usually don't have a compiler and won't generate errors before runtime. The advantages to a dynamic-typed language are that it often takes less time to initially write a program because it's less information to worry about. It also allows for more flexibility when writing code. For example, you could write a method that can take any type of data in its parameters much easier because a variable can be any type of object.

Static-typed languages are the opposite if dynamic-typed languages; they require every object to have a fixed type. Some static-typed languages include Java, C, and C++. In these languages, you have to specify a type whenever you declare a variable. You can't assign a string value to a variable that was previously an integer. The advantages of these types of languages are that it' easier to identify small bugs. The compiler will tell you about when and where you miss a semicolon or pass in the wrong data type. Another advantage is being able to point to any value in the code and know what data type it is. This makes it easier to loop through nested arrays because you can often tell where you are by looking at the variable's type.
